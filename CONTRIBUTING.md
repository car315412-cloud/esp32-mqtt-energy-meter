# Contributing

Thank you for your interest in improving this project.

## Before you contribute

- Keep the existing sketch structure and numbering unless there is a clear reason to change it.
- Avoid committing build artifacts from `build_*` folders.
- Do not include Wi-Fi passwords, MQTT credentials, tokens, or API keys in source files.
- Prefer small, focused changes that are easy to review.

## Suggested workflow

1. Fork or branch the repository.
2. Make your changes in one sketch or one documentation area at a time.
3. Test the sketch on the target board before opening a pull request.
4. Update `README.md` if you add a new module or change the project layout.

## Documentation style

- Use clear section headings.
- Keep file names and sketch names consistent with the existing numbering.
- Add images when they help explain wiring, flow, or the user interface.
